# 知识库项目常见问题 FAQ

## Q1: 知识库和传统文档管理系统有什么区别？

A: 传统 DMS 主要是文件存储 + 目录浏览；现代知识库的核心是**检索**（尤其是语义检索），用户不用关心文件在哪里、用关键词找不全也能用自然语言找。

## Q2: 100 人企业部署知识库需要多少预算？

A: 视需求深度。基础 SaaS 订阅：5-15 万 / 年。私有化 + 实施：30-80 万。含定制开发 + 数据迁移：80-200 万。

## Q3: 知识库能跟现有的 OA / ERP 集成吗？

A: 可以。常见集成点：SSO 单点登录、组织架构同步、文档回写、消息推送。技术方式：RESTful API / 消息队列 / 数据库直连。

## Q4: 文档安全怎么保证？

A: 三层防护：1）传输加密（HTTPS / TLS 1.3）；2）存储加密（AES-256）；3）权限控制（RBAC + 字段级 + 审计日志）。私有化部署时数据不出客户内网。

## Q5: 实施周期一般多长？

A: 100 人规模：4-8 周。1000 人规模：3-6 个月。10000 人规模：6-12 个月。周期差异主要在数据迁移和系统集成。

## Q6: 知识库和 LLM 大模型是什么关系？

A: 知识库是数据的"家"，LLM 是"理解者"。两者结合叫 RAG（检索增强生成）：先用知识库检索相关文档，再让 LLM 基于文档回答问题。既保证答案准确（基于实际文档），又保留 LLM 的理解能力。

## Q7: 上线后如何持续运营？

A: 三件事：1）内容运营：定期清理、补充、打标；2）用户运营：培训、激励、推广；3）技术运营：监控、扩容、安全更新。
